## Recommendations

1. **Monitor @napi-rs/keyring** — Set up GitHub watch for security advisories. The package is a Rust FFI binding (keyring-rs → Node.js via napi-rs). LongYinan is a well-known developer (napi-rs creator, voidzero team) which reduces but doesn't eliminate single-maintainer risk.

2. **Encrypted file fallback is good mitigation** — P98's AES-256-GCM encryption means even if keyring is compromised, tokens are still encrypted at rest via the file-based path.

3. **grammY has no SECURITY.md** — Consider opening a PR or issue to request one. grammY is well-maintained (3.5K stars, org-backed, active) but lacks a formal security reporting channel.

4. **No action needed on other deps** — axios (109K stars, org), chalk (23K, org), commander (28K), inquirer (21K), zod (42K), js-yaml (6.5K, org), ora (9.6K, sindresorhus), p-limit (2.8K, sindresorhus), MCP SDK (12K, org) are all well-maintained and organization-backed or by extremely well-known maintainers.
